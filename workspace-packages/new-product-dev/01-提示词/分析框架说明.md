# 分析框架说明 — 新品开发

本目的使用 6 个框架。下面写清**每个框架的内容 + 怎么用 + 在本链路哪步用**。维度/标签口径以 `05-标签字典/taxonomy.md` 为准。

---

## 1. 需求四象限 quad（定位空白机会 · 链路第 3 步）
- **内容**：横轴=满意度，纵轴=重要度。每个需求点按 VOC 出现频率估算重要度与满意度，落入四象限。
- **怎么用**：
  1. 从打标结果取每个需求点的提及频次 → 估重要度。
  2. 看现有产品/竞品对该需求的满足程度 → 估满意度。
  3. 落点：**高重要 × 低满意 = 创新突破口（P0）**；高重要 × 高满意 = 维持；低重要 × 低满意 = 放弃；低重要 × 高满意 = 差异化小品。
- **产出**：四象限图 + P0/P1 机会清单。

## 2. 创意罗盘 compass（生成概念 · 链路第 5 步）
- **内容**：将品类按「功能 / 外观 / 场景 / 情感 / 技术 / 叙事」等创意矿脉切分，组合生成产品概念。
- **怎么用**：对 P0 机会点，从 2–3 条矿脉交叉取灵感，长出概念方向；每个方向写清「解决了什么需求 + 差异点」。

## 3. TRIZ 40 原理 triz（结构化创新 · 链路第 5 步）
- **内容**：40 个发明原理（组合 / 分割 / 局部质量 / 预先作用 等）。
- **怎么用**：把 P0 机会抽象为技术矛盾，查对应原理，生成可落地的功能组合方案（如「静音+可伸缩」= 组合+分割）。

## 4. 领先用户 extreme（识别首波验证对象 · 链路第 4 步）
- **内容**：von Hippel 领先用户理论——找「走在使用前沿 + 自身受益大」的用户。
- **怎么用**：在 Reddit / 反馈中找有 DIY、非标、极端要求的表述，归为领先用户；其需求往往预示大众未来需求，作为概念首波验证对象。

## 5. JTBD jtbd（理解真实任务 · 链路第 2 步辅助）
- **内容**：用户「雇用」产品完成什么任务。
- **怎么用**：对高频痛点问「他真正要完成的任务是什么」，避免被表面功能误导。

## 6. 十三种需求 needs（合成价值主张 · 链路第 6 步）
- **内容**：基于马斯洛扩展的四层模型：身份 → 体验 → 功能 → 必买。
- **怎么用**：每个概念从四层自下而上合成价值主张（如「必买=宠物安全材质 → 功能=低噪 → 体验=家居装饰 → 身份=养宠品质生活」）。

---

## 链路串联
```
VOC打标 → 聚类未满足需求(JTBD辅助) → 需求四象限(quad) → 领先用户(extreme)
       → 概念方向(compass+TRIZ) → 价值主张(needs) → 套模板输出 → 校验清单核对
```
