# VOC 9维度五阶段打标工具

纯规则驱动的 VOC（用户之声）打标工具，支持 9 个维度的自动标签分类。
包含五阶段打标流程，准确率可达90%以上。

## 何时使用

- 用户上传 CSV 文件需要打标签
- 用户粘贴评价文本需要分类
- 用户提到"VOC打标""用户评价分类""标签体系"
- 需要快速批量处理大量评价数据
- 需要高精度标签（五阶段流水线模式）

## 两种打标模式

| 模式 | 特点 | 适用场景 |
|------|------|---------|
| 快速模式 | 纯规则引擎，本地运行，速度快 | 初筛、快速分类 |
| 流水线模式 | 五阶段流程，更高准确率 | 正式分析、深度洞察 |

## 五阶段打标流程

```
阶段一：粗匹配（高召回）
    ↓
阶段二：精确筛选（上下文验证+排除规则）
    ↓
阶段三：LLM抽样验证（多轮迭代，直到准确率≥80%）
    ↓
阶段四：调度器检查（逐条审核+补充遗漏）
    ↓
阶段五：二次验证（生成准确率报告）
```

### 阶段一：粗匹配（高召回）
- 使用关键词列表进行初步匹配
- 原则：宁多勿漏，确保高召回率
- 输出：初步标签结果

### 阶段二：精确筛选
- 应用上下文验证规则
- 应用排除规则（去除误匹配）
- 原则：去伪存真，提高准确率

### 阶段三：LLM抽样验证
- 随机抽样10%进行LLM验证
- 多轮迭代修正错误
- 直到抽样准确率≥80%停止

### 阶段四：调度器检查
- 逐条审核所有数据
- 修正错误标签
- 补充遗漏的标签

### 阶段五：二次验证
- 再次抽样验证
- 生成各维度准确率报告
- 输出最终结果

## 使用方法

### 1. 快速模式（纯规则）

```bash
python scripts/tagger.py input.csv --output output.csv --text-column "内容"
```

### 2. 流水线模式（五阶段，更高准确率）

```bash
python scripts/tagger.py input.csv --pipeline --output output.csv --text-column "内容"
```

### 3. 流水线模式 + LLM验证

```bash
python scripts/tagger.py input.csv --pipeline --use-llm --output output.csv --text-column "内容"
```

参数：
- `input.csv`：输入文件路径
- `--output`：输出文件路径（默认：输入文件名 + `_已打标`）
- `--text-column`：文本内容所在列名（默认自动检测"内容""评论""评价""正文"等）
- `--dimensions`：指定维度，逗号分隔（默认全部）
- `--pipeline`：启用五阶段流水线模式
- `--use-llm`：在流水线中使用LLM验证（需配置API）
- `--text`：单条文本测试
- `--list-dimensions`：列出所有维度和标签

## 输出格式

每个维度生成两列：
- `{维度}_标签`：命中的标签，多个用"、"分隔
- `{维度}_命中词`：命中的关键词原文，用于解释为什么打这个标签

## 准确率说明

| 模式 | 准确率 | 速度 |
|------|--------|------|
| 快速模式 | 70-80% | 快（本地规则） |
| 流水线模式（无LLM） | 85-90% | 中 |
| 流水线模式（含LLM） | 90-95% | 慢（需API调用） |

## 扩展规则

在 `scripts/rules/` 目录下修改对应维度的规则文件。
每个规则包含：
- `keywords`：关键词列表
- `context_patterns`：上下文验证正则（可选）
- `exclude_patterns`：排除模式（可选）
