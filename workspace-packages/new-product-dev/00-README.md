# 工作区方案包 · 新品开发 / 创新孵化

> 这是一个**自包含**的分析包。把它整包交给任意 AI，AI 即可基于包内真实数据完成「未满足需求挖掘 → 空白机会定位 → 新品概念生成」，并产出结构化分析包。

## 包内含什么

| 目录 / 文件 | 内容 | 给谁看 |
|---|---|---|
| `manifest.json` | 机器可读索引：目的、版本、源清单、框架、技能依赖、运行顺序 | AI 第一步读 |
| `00-README.md` | 本文件，人读总览 | 人 |
| `01-提示词/` | **SYSTEM.md（总提示词）+ 数据源说明 + 分析框架说明 + 技能调用说明** | AI |
| `02-数据包/` | 9 个源的 CSV（真实 VOC 样本 + 结构示例） | AI |
| `03-技能包/voc-tagger/` | VOC 九维打标技能 SKILL.md | AI |
| `04-交付模板/` | 输出模板 + 校验清单 | AI |
| `05-标签字典/taxonomy.md` | VOC 九维 + 十三种需求层次（打标口径） | AI |

## AI 运行顺序
1. 读 `manifest.json` → 2. 读 `01-提示词/SYSTEM.md` → 3. 载入 `02-数据包/*.csv` →
4. 按 `技能调用说明.md` 调 voc-tagger 打标 → 5. 按 `分析框架说明.md` 走链路 →
6. 套 `04-交付模板/输出模板.md` 产出 → 7. 跑 `校验清单.md` 核对。

## 关键约束
- **只用包内数据**，结论行级引用 `[(源文件#行号)]`，禁止编造 / 联网。
- 数据不足处标注「数据不足，建议补充」。
- s14 / s20 / s9 当前为结构示例，正式使用前请于主站拉取完整筛选集替换。

## 这套结构适用于所有「目的」
新品营销、活动运营、市场细分、竞品对标…… 每个目的工作区都按此三块（提示词 / 数据包 / 技能包）+ 补充（模板 / 校验 / 字典 / 索引）产出同构包。
