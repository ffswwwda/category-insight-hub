### 浮动栏"全页面显示"根因修复（commit 212698b，已推 origin/main）
- **根因**：`initFloatBar` 通过 `setView` 钩子维护 `_curView`，但**德国站/数据源直切路径**（`openDeDashboard` 等）直接操作 `.view` 的 `active` class、**绕过 `setView`**，导致 `_curView` 卡在 'Lab'，浮动栏在所有后续页面都残留显示。此前 Node 验证只测了走 setView 的页面，漏了绕过路径 → 用户实测"所有页面都有"。
- **修复（重写 initFloatBar + globFloatAction）**：
  - 不再维护 `_curView`；`update()` 直接从 DOM 读取当前激活视图：`document.querySelector('.view.active').id`
  - 仅 `viewLab` / `viewStress` 才显示；动态文案与 CTA 作用域按实际激活视图取
  - 用 **MutationObserver** 监听所有 `.view` 的 `class` 变化重新判定（涵盖一切绕过 setView 的切换，含德国站/数据源直切）
  - `globFloatAction()` 也改为从 DOM 读激活视图决定开哪个工作台
  - 移除 `setView` 里已失效的 `window.__onViewChange` 调用
- **验证（Node+jsdom）**：after init / Grid / **DE 绕过 setView** / Stress / Source 全部 `[HIDDEN]`；Lab+CTA 在视口下方 → `[VISIBLE]`(进入预测工作台)，Lab+CTA 进视口 → `[HIDDEN]`；globFloatAction 路由 Stress→virtual-user-stress-test / Lab+Grid→user-research-console。全部通过。

### AI办公全景 3D 重构完成（独立项目 ai-office-3d）
- **需求**：CSS 3D 效果和参考项目差距大，要求重建 3D 等轴测办公室
- **方案**：新建 `ai-office-3d/` 子项目，复刻 ai-office-react 技术栈（PixiJS v8 + React 19 + Vite 8 + TypeScript），但**去掉 Spine 依赖**（用程序绘制 fallback）

  - 我们的方案：纯 PixiJS Graphics API 绘制**地板网格 + 墙壁 + 柜子 + 绿植**（零图片依赖）
  - 桌子椅子用现有 `drawDeskFallback()` / `drawChairFallback()` 程序绘制
  - 角色用 `drawBody()` 程序绘制（圆头+身体+手臂+腿）

  - **去掉 desk/chair PNG**：`drawDeskFallback()` 增强版加显示器渐变+三色装饰线+小笔记本
  - **`loadOfficeAssets()` 加超时**：`Promise.race(Assets.load, 3s timeout)` 防止 404 挂起整个 init
